# Daydream — 像素眼睛专注计时器

一个轻量级 Windows 桌面小组件，用像素风格的小眼睛陪你专注工作。

```
┌─────────────────┐
│   👁      👁    │  ← 像素眼睛，每5分钟眨眼
│                 │
│    22:30        │  ← 大字倒计时
│    专注中…      │
│                 │
│  📊  ✕  ⚙      │  ← 统计 / 退出 / 设置
└─────────────────┘
```

---

## 功能特性

| 特性 | 说明 |
|------|------|
| ⏱ 自定义时长 | 1 – 120 分钟，点击 `−` / `+` 调整 |
| 👀 像素眼睛 | 每 5 分钟自动眨眼（闲置 / 专注 / 暂停均一致） |
| 👓 黑色椭圆眼镜 | 在设置中一键切换 |
| 🖱 悬浮控制 | 计时中将鼠标移到眼睛上，即可 **暂停 / 停止** |
| 🎉 鼓励语 | 完成后弹出自定义鼓励语（默认"你真棒！"）|
| 📊 今日统计 | 当天专注次数 + 累计时长，每天自动清零 |
| 📌 置顶悬浮 | 窗口始终在最前，不遮挡主要工作区 |
| 💾 记住位置 | 退出后下次在同一位置启动 |

## 快速开始

### 前提条件

- **Node.js** ≥ 18（[下载](https://nodejs.org/)）
- **npm** ≥ 9（随 Node.js 一起安装）

### 安装依赖并运行

```bash
# 进入项目目录
cd daydream

# 安装依赖（首次运行需要）
npm install

# 启动应用
npm start
```

### 打包为 Windows 安装包（可选）

```bash
npm run build
```

打包产物在 `dist/` 文件夹中，生成 `.exe` 安装程序。

> **注意**：打包需要在 Windows 上执行，且需要网络（下载 Electron 二进制）。

## 使用说明

### 基本流程

1. 启动后小组件会出现在屏幕右侧（上次的位置）。
2. 点击 `−` / `+` 设定专注时长。
3. 点击 **开始专注**，计时开始，眼睛进入"盯着你"模式。
4. 专注中如需操作：  
   - 将鼠标移到眼睛区域 → 出现 `⏸` 和 `⏹` 按钮。  
   - `⏸` 暂停计时 / 再次点击恢复。  
   - `⏹` 终止计时并返回主界面。
5. 时间到后显示鼓励语与本次统计，点击 **再来一次** 继续。

### 移动位置

在窗口任意空白处按住鼠标左键拖动即可。重启后自动恢复到上次位置。

### 设置（⚙ 按钮）

| 选项 | 说明 |
|------|------|
| 鼓励语 | 计时结束时显示的文字，最多 12 字 |
| 戴眼镜 | 为像素眼睛戴上黑色椭圆眼镜 |
| 默认时长 | 下次打开应用时的预设分钟数 |

### 统计（📊 按钮）

显示当天：
- 完成专注 **次数**
- 累计专注 **分钟数**

> 统计每天零点自动重置。

## 项目结构

```
daydream/
├── main.js          # Electron 主进程（窗口管理、IPC、数据持久化）
├── preload.js       # 安全上下文桥（仅暴露必要 API 给渲染进程）
├── lib/
│   └── focus-db.js  # 今日专注统计 JSON 存储（轻量「本地库」）
├── package.json
├── data/            # 开发时专注数据目录（已 .gitignore，自动生成）
├── assets/
│   ├── icon.png     # 任务栏 / 开发用图标（256×256，灰白底 + 黑框像素眼镜）
│   └── icon.ico     # Windows 安装包用图标（由脚本生成）
├── scripts/
│   └── generate-icon.js
└── src/
    ├── index.html   # 窗口 HTML 结构
    ├── styles.css   # 中性灰 UI + 像素眼睛样式
    └── renderer.js  # 渲染进程：Canvas 眼睛绘制、计时器、事件处理
```

### 关键设计决策

- **`contextIsolation: true` + preload**：遵循 Electron 安全最佳实践，渲染进程无法直接访问 Node.js API。
- **`electron-store`**：窗口位置、设置等写入 `%APPDATA%\daydream\config.json`。
- **专注统计 JSON**：`lib/focus-db.js` 写入 `data/focus-data.json`（开发，已 gitignore）或打包后 `%APPDATA%\daydream\focus-data.json`；按本地日期 `YYYY-MM-DD` 汇总，并保留最近若干条单次记录，无需 SQL。
- **Canvas 像素画**：所有眼睛图形由 5×5 像素块拼成的二维数组驱动，方便扩展新皮肤或表情。
- **眨眼节奏**：统一为每 5 分钟一次（闲置、专注、暂停状态相同）。
- **`-webkit-app-region: drag`**：整个卡片均可拖动，交互元素用 `.no-drag` 类排除。
- **应用图标**：`npm run icon` 根据与主界面一致的像素眼 + 黑框眼镜重新生成 `assets/icon.png` 与 `icon.ico`（浅灰白背景 `#ececec`）。

## 常见问题

**Q：窗口背景变成黑色不透明？**  
确保 `transparent: true` 生效，某些 Windows 11 系统需要在"系统 → 显示 → 透明效果"中开启透明效果。

**Q：字体显示成方块？**  
应用需要联网加载 Google Fonts (`Press Start 2P`)。首次使用有网络时会自动缓存；若离线，自动回退到系统字体，功能不受影响。

**Q：如何修改眼睛颜色 / 皮肤？**  
打开 `src/renderer.js`，修改 `PAL` 数组中的颜色值，或修改 `FRAME` / `GLASSES` 像素数组来改变形状。
